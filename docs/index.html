Aroma-Distribuido

Plataforma de e-commerce de perfumería de lujo implementada como sistema distribuido con Redis, RabbitMQ, Worker, Docker y observabilidad completa.

🚀 Inicio Rápido GitHub Swagger API

Servicios del Sistema

Todos los servicios desplegados con Docker Compose

⚡

Frontend Vue 3

Puerto 80 · Nginx

SPA con Pinia, Vue Router, TailwindCSS y soporte bilingüe ES/EN.

🐍

Backend FastAPI

Puerto 8000 · Uvicorn

14 módulos de API REST, JWT auth, middleware de Request-ID y logs JSON.

🐘

PostgreSQL 15

Puerto 5432

16 tablas relacionales con Alembic migrations. Datos persistentes con volumen Docker.

⚡

Redis 7

Puerto 6379 · LRU 256MB

Caché de productos, categorías, búsquedas y destacados. Invalidación automática.

🐰

RabbitMQ 3.13

Puerto 5672 · Mgmt 15672

Exchange topic aroma_events con 4 queues durables para eventos.

⚙️

Worker Python

Background service

Consume mensajes de RabbitMQ: emails, notificaciones, inventario. Auto-reconexión.

📋

Dozzle

Puerto 8888

Visualización en tiempo real de logs de todos los contenedores del sistema.

🐳

Portainer

Puerto 9000

Administración visual de Docker: contenedores, volúmenes, redes e imágenes.

Estrategia de Caché — Redis

Patrón de Clave Contenido TTL Invalidación
products:list:* Listas paginadas con filtros 5 min Al crear/editar/eliminar producto
products:detail:{id} Detalle completo del producto 5 min Al actualizar o eliminar ese producto
products:featured:* Productos destacados 2 min Al modificar cualquier producto
search:* Resultados de búsqueda y sugerencias 3 min Al modificar cualquier producto
categories:* Listas de categorías 10 min Al modificar categorías

Redis — Caché

8 servicios: frontend, backend, db, redis, rabbitmq, worker, dozzle, portainer.

Observabilidad Básica

Variables de Entorno

README Completo

Documentación con quickstart, arquitectura, variables de entorno, y guía de uso.

Simulación de Fallos

Endpoints en /api/v1/simulate/* para demostrar resiliencia de Redis, RabbitMQ y Worker.

Inicio Rápido

Levanta todo el sistema con un solo comando

Requisitos

Docker Desktop Docker Compose v2 Git 8 GB RAM mínimo
# 1. Clonar repositorio
git clone https://github.com/isjuan99/Programacion-Distribuidad-2026.git
cd aroma-distribuido

# 2. Levantar todos los servicios
docker compose up -d

# 3. Verificar que todos los contenedores estén corriendo
docker compose ps

# 4. Crear administrador inicial
docker compose exec backend python create_admin.py

Variables de Entorno

# === Base de Datos ===
DATABASE_URL=postgresql://aroma_user:aroma_secret@db:5432/aroma_db

# === Redis ===
REDIS_URL=redis://redis:6379/0
CACHE_TTL_PRODUCTS=300
CACHE_TTL_CATEGORIES=600
CACHE_TTL_FEATURED=120
CACHE_TTL_SEARCH=180

# === RabbitMQ ===
RABBITMQ_URL=amqp://aroma_user:aroma_secret@rabbitmq:5672/
RABBITMQ_EXCHANGE=aroma_events

# === Servicio ===
SERVICE_NAME=aroma-backend
ENVIRONMENT=production
